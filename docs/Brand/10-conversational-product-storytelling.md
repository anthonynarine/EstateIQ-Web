# Conversational Product Storytelling

EstateIQ should sound like a knowledgeable person explaining the product to a small landlord sitting across the table.

The primary rule:

Write as if you're explaining EstateIQ to a customer sitting across the table from you.

Never write as though you are explaining the system to another software engineer.

## Conversation First

Every section should answer a customer question.

Instead of:

```text
Document Intelligence
```

Prefer:

```text
Your documents should work for you.
```

Instead of:

```text
Ledger-first Accounting
```

Prefer:

```text
Wondering where your money actually went?
```

Instead of:

```text
Grounded AI
```

Prefer:

```text
AI is only useful if it understands your business.
```

Instead of:

```text
Deterministic Reports
```

Prefer:

```text
Reports you can actually trust.
```

## Natural Questions

When appropriate, introduce sections using questions customers actually ask:

- Where did all my money go?
- What does this mortgage statement mean?
- Which building is costing me the most?
- Did this tenant actually pay?
- Why were expenses higher this month?
- Where is this invoice connected?

The page should feel like it is answering these questions.

## Explain Like A Human

Do not remove technical depth. Change the order.

Every major section should follow:

1. Simple outcome
2. Practical explanation
3. System visualization
4. Technical explanation
5. Call to action

## Reward Curiosity

Technical concepts should remain:

- ledger-first accounting
- grounded AI
- Gap Tool
- DANA
- document intelligence
- deterministic reporting
- capability registry

But they should appear after readers understand why they matter.

## Never Assume Technical Knowledge

Do not assume customers know terms like:

- deterministic
- orchestration
- selector
- capability
- ledger-first

Explain those concepts naturally before using them heavily.

## Tone

EstateIQ should sound:

- confident
- calm
- helpful
- knowledgeable
- honest
- human

Never:

- overly academic
- buzzword-heavy
- salesy
- hype-driven

## Customer Test

Every paragraph should pass this test:

"If I read this aloud to a landlord sitting across from me, would it sound natural?"

If not, rewrite it.

## Accuracy Rules

Never claim:

- self-learning AI
- autonomous capability creation
- documents are financial truth
- unsupported OCR
- unsupported automation

Always preserve:

- confirmed records and ledger data remain financial truth
- documents are connected evidence and supported context
- AI explains records, reports, and supported documents
- new AI coverage comes from reviewed deterministic capabilities
- unsupported questions should not become guessed answers
